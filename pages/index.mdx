# Introdução

Bem-vindo à documentação da API de Auditoria da TOTVS. Esta API foi desenvolvida para proporcionar uma solução eficaz para a verificação de acessos a sistemas dentro do ambiente TOTVS, com foco específico na identificação de funcionários que foram demitidos, mas que ainda mantêm acesso a algum sistema da TOTVS.

## Objetivo

A API de Auditoria da TOTVS tem como objetivo garantir a conformidade e a segurança dos sistemas, fornecendo uma maneira robusta de auditar e monitorar acessos. Com esta API, você poderá:

- **Verificar o Status de Acesso**: Identificar todos os funcionários que foram desligados, mas que ainda possuem acesso ativo a qualquer sistema da TOTVS.
- **Garantir Conformidade**: Assegurar que os acessos sejam revogados corretamente após a demissão, atendendo às políticas de segurança e conformidade da empresa.
- **Prevenir Riscos de Segurança**: Reduzir os riscos de segurança associados a acessos não autorizados por parte de ex-funcionários.

## Funcionalidades

- **Consulta de Funcionários Demitidos**: Permite consultar uma lista de funcionários que foram demitidos e verificar os sistemas aos quais ainda têm acesso.
- **Relatórios Detalhados**: Geração de relatórios sobre acessos pendentes para facilitar a gestão e auditoria.
- **Atualizações de Acesso**: Notificações sobre acessos que precisam ser revogados para garantir que a conformidade seja mantida.

## Público-Alvo

Esta documentação é destinada a administradores de sistemas e equipes de auditoria que necessitam monitorar e gerenciar acessos dentro dos sistemas da TOTVS. Fornece detalhes sobre os endpoints da API, exemplos de requisições e respostas, e orientações para a integração com os sistemas existentes.
